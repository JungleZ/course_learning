# 🚌 等公交互动聊天室功能说明

## 🎯 功能概述

这是一个创新的社交功能，让用户在等公交车时可以通过Web端进行实时互动聊天，在等待的过程中也能遇见缘分、传递温暖。

---

## ✨ 核心功能

### 1. **实时在线聊天**
- ✅ 匿名加入聊天室，保护隐私
- ✅ 实时消息收发，2秒轮询更新
- ✅ 显示在线人数
- ✅ 系统消息提示（加入/离开）

### 2. **公交车到站模拟**
- ✅ 每2分钟一班公交车
- ✅ 实时倒计时显示
- ✅ 延误提示（随机生成延误原因）
- ✅ 公交车即将到站提醒

### 3. **聊天时长追踪**
- ✅ 实时显示聊天时长
- ✅ 满5分钟后可加好友
- ✅ "可加好友"徽章提示

### 4. **智能状态提示**
- ✅ 无人在线时的温馨提示
- ✅ 公交车延误原因说明
- ✅ 文明交流提醒

---

## 🚀 使用流程

### 步骤1: 进入聊天室
从主页点击"等公交聊天室"入口卡片或直接访问：
```
http://localhost:5000/bus-chat
```

### 步骤2: 设置昵称
- 输入一个有趣的昵称（最多20个字符）
- 点击"加入聊天室"按钮
- 系统会分配唯一的用户ID

### 步骤3: 开始聊天
- 在输入框中输入消息
- 按Enter或点击发送按钮
- 消息会立即显示在聊天区域

### 步骤4: 查看公交车状态
- 顶部显示下一班公交车的倒计时
- 如果延误，会显示延误原因
- 公交车即将到站时会有提示

### 步骤5: 等待5分钟加好友
- 聊天满5分钟后，右上角会出现"可加好友"徽章
- 此时可以与其他用户交换联系方式
- 继续聊天或离开聊天室

### 步骤6: 离开聊天室
- 点击"离开"按钮
- 或直接关闭浏览器窗口
- 系统会自动记录离开消息

## 🔧 技术实现

### 后端API

#### 1. 加入聊天室
```
POST /api/chat/join
Body: { "username": "用户昵称" }
Response: { 
    "success": true,
    "user_id": "唯一ID",
    "username": "昵称",
    "online_count": 在线人数,
    "messages": [历史消息],
    "chat_start_time": 聊天开始时间
}
```

#### 2. 发送消息
```
POST /api/chat/send
Body: { "user_id": "用户ID", "message": "消息内容" }
Response: { "success": true }
```

#### 3. 获取消息
```
GET /api/chat/messages?user_id=xxx
Response: {
    "messages": [消息列表],
    "online_count": 在线人数,
    "chat_duration": 聊天时长(秒),
    "can_add_friend": 是否可加好友
}
```

#### 4. 公交车状态
```
GET /api/chat/bus-status
Response: {
    "next_bus_in": 倒计时(分钟),
    "is_delayed": 是否延误,
    "delay_reason": "延误原因",
    "scheduled_interval": 2,
    "online_users": 在线人数
}
```

#### 5. 离开聊天室
```
POST /api/chat/leave
Body: { "user_id": "用户ID" }
Response: { "success": true }
```

### 前端技术
- **轮询机制**: 每2秒获取一次最新消息
- **自动清理**: 超过2分钟未活动的用户自动移除
- **本地存储**: 暂不使用，所有数据在内存中
- **页面关闭**: 使用`beforeunload`事件自动离开

1. 打开网页，加入聊天室

**收获**: 即使独自一人，也知道有人在同样的情境中

## 💡 设计理念

### 1. **温暖相遇**
在城市的快节奏生活中，等公交是一个短暂的停顿。我们希望通过这个聊天室，让这段等待时间变得有意义，让人们有机会相遇、交流、分享。

### 2. **低压社交**
- 匿名参与，无需注册
- 随时可以离开，没有负担
- 5分钟门槛，避免骚扰
- 文明交流，传递正能量

### 3. **情境共鸣**
大家都在等同一班公交车，有共同的话题和情境。这种共同的体验更容易拉近距离，产生共鸣。

### 4. **时间价值**
- 2分钟一班的频率，不会等太久
- 5分钟的聊天时长，足够初步了解
- 既不会太短（无法深入），也不会太长（造成负担）

## 🔒 隐私与安全

### 隐私保护
- ✅ 完全匿名，不需要真实身份
- ✅ 不保存聊天记录（刷新即清空）
- ✅ 不收集个人信息
- ✅ 用户ID临时生成，关闭即失效

### 安全措施
- ✅ 消息长度限制（200字符）
- ✅ HTML转义，防止XSS攻击
- ✅ 输入验证，防止SQL注入
- ✅ 自动清理不活跃用户

### 文明公约
- 尊重他人，礼貌交流
- 不发布违法、暴力、色情内容
- 不泄露个人隐私信息
- 不发送广告或垃圾信息
