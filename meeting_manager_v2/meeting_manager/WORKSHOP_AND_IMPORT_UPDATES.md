# 功能更新说明 - Workshop Toggle & 接龙导入

## 📋 更新内容

### 1. 工作坊开关按钮功能 ✨

#### 功能描述
在创建/编辑会议时，添加了**开关按钮**来控制是否启用工作坊功能。

#### 实现方式
- **开关状态**：使用 Bootstrap switch 组件，打开/关闭工作坊
- **主讲人字段**：开关打开时必须填写主讲人姓名
- **显示控制**：
  - 开关关闭 → 议程表中不显示工作坊阶段
  - 开关打开 + 填写主讲人 → 议程表中显示工作坊（独立阶段）

#### 用户操作流程
1. 创建/编辑会议
2. 找到"工作坊（可选）"部分
3. 点击开关按钮启用/禁用工作坊
4. 启用后填写：
   - 主讲人（必填）
   - 工作坊名称（中/英文）
   - 工作坊时长（默认30分钟）

#### 数据库变更
- 新增字段：`workshop_speaker TEXT`

---

### 2. 微信群接龙导入功能 🚀

#### 功能描述
新增功能：**从微信群接龙帖子自动解析并导入报名记录**，不影响现有功能。

#### 实现方式
- **新增路由**：`/meeting/<meeting_db_id>/import-signup`
- **解析函数**：`parse_wechat_signup(text)` 支持多种帖子格式
- **页面入口**：会议详情页添加"导入报名"按钮

#### 支持的帖子格式

**格式1：标准英文**
```
MM: Guiling/Sibley
SAA:
Opening remark: 
TOM: 
Timer:
Photographer: Sophie
Ah-counter: Wendy
Speaker 1: Liddy
Speaker 2: Cathy
IE 1: 
IE 2: 
```

**格式2：中文**
```
主持人: Guiling
迎宾官: Sibley
计时官: Wendy
摄影师: Sophie
备稿演讲1: Liddy
备稿演讲2: Cathy
个评1: 
个评2: 
```

**格式3：混合格式**
```
MM: Guiling/Sibley
Timer: Wendy
Photographer: Sophie
Speaker 1: Liddy
Speaker 2: Cathy
Speaker 3: Sibley
Speaker 4: Xiaoshu
IE 1: 
IE 2: 
```

#### 角色映射规则
系统自动识别并映射以下角色：
- **主持团队**：MM、SAA、Opening remark
- **执行团队**：TOM、Timer、Game master、Photographer、Ah-counter、Grammarian、GE
- **即兴演讲**：TTM、TTE
- **备稿演讲**：Speaker 1-4
- **个人点评**：IE 1-4

#### 用户操作流程
1. 进入会议详情页
2. 点击"导入报名"按钮
3. 粘贴微信群接龙帖子内容到文本框
4. 点击"导入报名"按钮
5. 系统自动解析并显示导入成功的记录数

#### 特性
- ✅ 自动跳过空角色（未填写姓名的角色）
- ✅ 自动跳过重复报名记录
- ✅ 支持多个人报名同一个角色（用 `/` 分隔）
- ✅ 支持中英文角色名称识别
- ✅ 不影响现有手动报名功能

##  使用说明

### 工作坊使用
1. 创建/编辑会议时，找到"工作坊（可选）"部分
2. 点击开关按钮启用工作坊
3. 填写主讲人姓名（必填）
4. 填写工作坊名称和时长（可选）
5. 保存后，议程表中会自动显示工作坊阶段

### 接龙导入使用
1. 从微信群复制接龙帖子内容
2. 进入会议详情页，点击"导入报名"按钮
3. 粘贴接龙内容到文本框
4. 点击"导入报名"按钮
5. 系统自动解析并导入有效的报名记录

---

## ️ 注意事项

1. **工作坊主讲人必填**：开关打开时必须填写主讲人，否则工作坊不会显示在议程中
2. **接龙格式识别**：系统会自动识别角色名称，建议保持标准的 `角色: 姓名` 格式
3. **重复报名处理**：同一个人报名同一个角色多次，系统会自动去重
4. **不影响现有功能**：接龙导入是新增功能，不影响现有的手动报名方式

## 🔧 技术细节

### 角色映射表
系统内置了完整的角色名称映射，支持：
- 英文缩写（MM、SAA、TOM、TTM、TTE、GE、IE）
- 英文全称（Master of Ceremonies、Timer、Grammarian 等）
- 中文名称（主持人、计时官、语法官等）
- 混合写法自动识别

### 解析算法
1. 逐行扫描接龙文本
2. 使用正则表达式匹配 `角色: 姓名` 格式
3. 通过角色映射表转换为标准角色名称
4. 处理多人报名（`/` 分隔）
5. 过滤空值和无效数据
6. 批量插入数据库（去重处理）
