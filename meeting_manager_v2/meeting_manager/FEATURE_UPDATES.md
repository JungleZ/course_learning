# Meeting Manager 功能更新说明

## 更新日期：2026-05-02

## 一、默认 President 角色分配

### 修改内容
在闭幕环节（Closing Phase）中，以下三个角色默认分配给 **Bass**（President），无需报名：

1. **嘉宾分享 (Guest Sharing)** - 3分钟
2. **颁奖环节 (Awarding Time)** - 3分钟  
3. **闭幕词 (Closing Remarks)** - 2分钟

### 实现方式
- 在 `PHASE_TEMPLATES` 的 closing 阶段添加了 `"default_member": "Bass"` 属性
- 在议程生成逻辑中，如果角色无人报名且设置了 `default_member`，则自动显示为 "Bass"
- 这些角色仍可被其他人报名覆盖

## 二、工作坊 (Workshop) 功能

### 新增字段
会议信息表新增三个字段：
- `workshop_zh` - 工作坊名称（中文）
- `workshop_en` - 工作坊名称（英文）
- `workshop_duration` - 工作坊时长（分钟），默认30分钟

### 功能特点
1. **可选功能**：工作坊为可选字段，不填写不会显示
2. **议程集成**：工作坊会自动插入到闭幕阶段的**倒数第二个位置**（在角色预订和会议结束之前）
3. **时间融合**：工作坊时间会自动与上下环节衔接，显示正确的时间点
4. **多语言支持**：支持中文、英文和中英双语三种显示模式
5. **时长可配置**：可在创建/编辑会议时设置工作坊时长（10-120分钟）

### 使用方式
1. 创建或编辑会议时，在表单底部找到"工作坊"区域
2. 填写工作坊名称（中/英文）
3. 设置工作坊时长（默认30分钟）
4. 生成议程时，工作坊会自动出现在议程表末尾

## 四、数据库更新

### 自动迁移
应用启动时会自动检测并添加以下新字段到 meetings 表：
- `workshop_zh` (TEXT)
- `workshop_en` (TEXT)  
- `workshop_duration` (INTEGER DEFAULT 30)

### 兼容性
- 现有会议数据不受影响
- 新字段对于旧会议默认为空/默认值
- 无需手动执行数据库迁移脚本
